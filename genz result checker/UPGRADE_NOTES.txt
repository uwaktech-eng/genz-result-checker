RESULT CHECKER UPGRADE NOTES

Main upgrades applied
- Backend rebuilt to match the richer admin/student UI already present in the project.
- Stronger security: browser-bound sessions, request freshness nonce + timestamp checks, login/reset rate limiting, salted password hashing with automatic upgrade from old hashes, audit logging, anti-embed protection, and admin-only export routes.
- Full admin actions wired: bootstrap, create sub-admin, student create/update, student/result state changes, CSV imports/exports, ranking recalculation, bulk email, SMS contact export, password reset, and student result loading by exam code.
- Result builder model improved: per-subject rows with exam code, session, term, publish state, view-active state, published date, signature URL, ranking, and pass/fail status.
- Public portal tightened: student signup is disabled by default, admin public signup is disabled by default, and both can be controlled from settings.
